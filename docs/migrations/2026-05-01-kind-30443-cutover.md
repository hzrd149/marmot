# KeyPackage Cutover: `kind:443` to `kind:30443`

This document records the Marmot rollout plan for migrating KeyPackages from legacy `kind:443` events to addressable `kind:30443` events.

## Cutover Date

The Marmot project cutover date is **May 1, 2026, at 00:00:00 UTC**.

Before that date, deployments that need interoperability with legacy clients SHOULD follow the transition behavior below. On and after that date, the Marmot rollout expects `kind:30443` to be the only KeyPackage format used for new publication and discovery.

## Before May 1, 2026, at 00:00:00 UTC

Clients that need interoperability with legacy peers SHOULD:

- Query for both `kind:443` and `kind:30443` KeyPackages
- Prefer valid `kind:30443` KeyPackages when both are available
- Fall back to the freshest valid `kind:443` KeyPackage only when no valid `kind:30443` is available
- Dual-publish equivalent KeyPackages as both `kind:30443` and `kind:443`

When dual-publishing:

- The `kind:30443` event is the canonical event for the new specification
- The legacy `kind:443` event SHOULD carry the same KeyPackage content and the same applicable tags, except for the `d` tag, which `kind:443` does not support
- Clients SHOULD continue rotating the `kind:30443` event under the same `d` tag as described in [MIP-00](../../00.md)

## On and After May 1, 2026, at 00:00:00 UTC

Clients SHOULD:

- Query only `kind:30443` for KeyPackage discovery
- Publish only `kind:30443` KeyPackage events
- Stop publishing new `kind:443` KeyPackage events

Legacy `kind:443` support becomes optional after the cutover. Deployments may keep temporary compatibility shims, but the Marmot rollout no longer requires them after this date.

## Legacy Cleanup

Because `kind:443` events are not addressable, they cannot be replaced in place. After the cutover:

- Deployments SHOULD send NIP-09 deletion events for legacy `kind:443` KeyPackages once equivalent `kind:30443` replacements are available
- Deployments SHOULD communicate the cutover to implementers and users through release notes, changelog entries, and other project channels
